2.构造函数的调用顺序问题

#include <iostream>
#include <string>
using namespace std;

class A{
    mutable int a;
    public:
        A(){
            cout << "A's Constructor called \n";
        }
        ~A(){
            cout << "A's Destructor called \n";
        }
};

class B{
    A a;
    public:
        B(){
            cout << "B's Constructor called \n";
        }
        ~B(){
            cout << "B's Destructor called \n";
        }
};

int main(int argc, char const *argv[]){
    B b1;
}

/*
A's Constructor called

B's Constructor called

B's Destructor called

A's Destructor called
*/

B类中含有A类，先按照B类的参数定义顺序来定义成员数据，再调用构造函数来初始化成员数据，所以先调用A的构造函数，再调用B的构造函数。

析构函数则是由外到内调用的。

或者按照定义顺序：

设已经有A,B,C,D4个类的定义，程序中A,B,C,D析构函数调用顺序为？
C c;
void main()
{ 
  A*pa=new A(); 
  B b; 
  static D d; 
  delete pa; 
}
  1. 首先，由于 c 是一个自动对象，其析构函数会在它所在的作用域结束时自动调用。(整个程序的最后)
  2. 接下来，由于 pa 是通过 new 运算符动态分配得到的，需要使用 delete 运算符手动释放内存。因此，在 delete pa; 语句执行时，会先调用 A 类的析构函数，再释放内存。
  3. 最后，在程序结束时，会先调用 B 类的析构函数，然后调用 D 类的析构函数。

综上所述，A、B、C、D 类的析构函数调用顺序为：A 的析构函数 -> B 的析构函数 -> D 的析构函数 -> c 的析构函数 。

请注意，静态对象的析构函数在程序结束时调用，而不是在其所在的函数结束时调用。静态对象的生命周期从程序开始到结束，析构函数的调用顺序与它们的定义顺序保持一致。

static类对象只有当main函数结束了才会调用析构函数。

2.下面哪种情况会发生复制省略？

A.函数返回一个临时对象

B.函数参数是一个右值引用

C.初始化一个对象时使用同类型的另一个对象

D.上述都是（D）

复制省略（copy elision）是编译器的一种优化技术，在特定情况下可以避免对象的拷贝或移动操作，从而提高程序的性能和效率。以下是几种情况可能会触发复制省略：



返回值优化（RVO）：

当函数返回一个对象时，如果被返回的对象是一个局部对象，且该对象没有被其他代码引用，编译器可以将这个局部对象直接构造在调用者提供的存储空间中，而不进行实际的拷贝或移动操作。



命名返回值优化（NRVO）：

类似于返回值优化，当函数返回一个对象时，如果被返回的对象是一个局部对象，并且该对象在函数内部被命名并声明为函数返回值，编译器可以直接将这个局部对象构造在函数外部的目标对象中，而不进行实际的拷贝或移动操作。



参数传递优化：

当将一个对象作为函数参数进行传递时，如果传递的对象是一个临时对象，并且该临时对象没有被其他代码引用，编译器可以直接将这个临时对象构造在函数内部的目标对象中，而不进行实际的拷贝或移动操作。

3.下面哪种写法可以利用复制省略提高性能？

A.std::string s = "hello";

B.std::string s("hello");

C.std::string s = std::string("hello");

D.上述都是 （B）

在上述选项中，选项 B 可以利用复制省略（copy elision）提高性能。

复制省略是一种编译器的优化技术，在某些情况下，可以避免不必要的对象拷贝或移动操作，从而提高程序的性能和效率。

在选项 B 中，使用直接初始化语法创建 std::string 对象 s，并将 “hello” 作为参数传递给构造函数。由于这里只涉及一次对象的直接初始化，编译器可以通过复制省略来避免额外的拷贝操作。

在其他两个选项中，都使用了赋值操作符，会导致初始化过程中发生一次拷贝操作。尽管现代编译器通常可以进行 RVO（返回值优化）或 NRVO（命名返回值优化）等优化来减少对象拷贝的开销，但仍然不如直接初始化语法在性能上高效。

因此，在这个特定的情况下，选项 B 的写法可以通过复制省略提高性能。

const #

1.const类型的对象只能调用const成员函数

2.开发者不可以将构造函数设置const成员函数

3.当一个对象按值传递给函数时，函数会在栈上创建一个新的对象，并将原始对象的值复制到这个新对象中。函数在执行过程中对该副本进行操作，而不会影响原始对象。

4.定义const对象

在 C++ 中，const 也可以用来修饰对象，称为常对象。一旦将对象定义为常对象之后，就只能调用类的 const 成员（包括 const 成员变量和 const 成员函数）了。



定义常对象的语法和定义常量的语法类似：

const  class  object(params);

class const object(params);



当然你也可以定义 const 指针：

const class *p = new class(params);

class const *p = new class(params);



class为类名，object为对象名，params为实参列表，p为指针名。两种方式定义出来的对象都是常对象。

5.

const成员函数是指在C++中，被声明为const的成员函数。const成员函数在类内部或类外部定义时使用const关键字进行标识，并且在函数签名的末尾加上const修饰符。



const成员函数有以下特点：



不修改对象状态：const成员函数承诺不会修改类的数据成员。它们可以读取对象的数据成员和调用其他const成员函数，但不能修改对象的任何非mutable成员。



保护对象的常量性：const成员函数在被调用时，如果对象为const或通过const引用或指针调用，则只能调用const成员函数。



成员函数重载：同名的非const成员函数和const成员函数可以共存，并根据调用对象的常量性来确定具体调用哪个函数。



常量对象调用：当使用常量对象调用成员函数时，只能调用const成员函数。这样可以确保常量对象的状态不会被修改。



通过在成员函数的声明和定义中添加const关键字，可以将其附加的常量属性表达出来，在编译器层面进行类型检查和限制，提供更安全和可靠的代码编写方式。

6.同一个程序中的一个函数可以同时有const和non-const两个版本。这种情况下，这两个版本被视为函数重载，根据调用时的对象是否为const对象来选择合适的版本进行调用。

考虑以下示例：

class MyClass {

public:

    void function() {

        // 非常规范的函数实现

    }

    

    void function() const {

         // 常规范的函数实现

    }

};

上述代码定义了一个名为function的成员函数，并分别提供了一个非const版本和一个const版本。非const版本用于对非const对象进行操作，而const版本用于对const对象进行操作。

当非const对象调用该函数时，将会调用非const版本的函数；当const对象调用该函数时，将会调用const版本的函数。通过这种方式，可以实现对不同类型（const和非const）的对象进行特定的操作，并保证不改变const对象的状态。

使用const和非const版本的函数重载，可以提高程序的灵活性和可读性，在不改变对象状态的情况下，可以对const对象进行操作，避免意外修改对象的行为。

7.如果在一个类中声明以下4个重载函数，有哪两个是互相冲突的？

Point fun1();  // 1
const Point fun1();  // 2
Point fun1() const;  // 3
const Point fun1() const;  // 4

（1与2冲突，3与4冲突）

这是因为函数重载需要根据函数的参数列表和函数的顶层 const（对于非成员函数而言）来进行区分，但不考虑函数的返回类型。只有函数的返回类型不同时，它们会被视为相同的函数，产生冲突。所以1与2相冲突。

类中存在后置const的常函数，与普通类成员函数之间可以实现重载。

注：后置const作用在与this指针上，使其原本的 type * const this 指针转变为 const type * const this指针，因此在声明为后置const的常对象中不可修改类成员变量（mutable与const_cast除外）。

禁用成员函数的使用。这是通过附加= delete说明符到该函数声明的结尾来完成的。删除的函数是隐式内联的，删除的函数定义必须是函数的第一个声明。

explicitly deleted

禁用拷贝构造函数：

class A { 
public: 
    A(int x): m(x) { } 
      
    // Delete the copy constructor 
    A(const A&) = delete;      
    // Delete the copy assignment operator 
    A& operator=(const A&) = delete;  
    int m; 
};

禁用不需要的参数转换:

class A { 
public: 
    A(int) {} 

    // Declare the conversion constructor as a  deleted function. Without this step,  
    // even though A(double) isn't defined,  the A(int) would accept any double value
    //  for it's argumentand convert it to an int 
    A(double) = delete;  
};

初始化成员列表 #

成员是按照他们在类中出现的顺序进行初始化的，而不是按照他们在初始化列表出现的顺序初始化的，看代码。

class foo{
    int i ;
    int j ;
    foo(int x):j(x), i(j){} // i值未定义
};

使用初始化列表除了可以提升程序性能外，还可以完成一些使用常规构造函数无法完成的功能。

通过初始化列表，可以在对象的构造过程中直接对成员变量进行初始化。这种方式可以避免在构造函数体内先执行默认初始化，然后再赋值的操作，提高了效率。

此外，使用初始化列表还可以完成以下功能：

  1. 初始化常量成员变量：类中的常量成员变量必须在初始化列表中进行初始化，因为它们不能通过赋值语句来修改。
  2. 初始化引用成员变量：引用成员变量必须在对象构造时进行初始化，并且无法在构造函数体内修改引用目标。因此，只能在初始化列表中对引用成员变量进行初始化。
  3. 调用基类的构造函数：派生类的构造函数在调用之前会自动调用基类的构造函数。通过初始化列表，可以明确指定要调用的基类构造函数以及传递给基类构造函数的参数。
  4. 对象成员变量的初始化：在类中包含其他类对象作为成员变量时，可以使用初始化列表来初始化这些成员对象。

总而言之，初始化列表不仅可以提升程序性能，还可以在构造对象时实现特定的初始化需求，如初始化常量或引用成员变量、调用基类构造函数等。

class Test {
public:
    int first;
    int second;
    Test(int value): second(value), first(second) {}
};

int main() {
    Test t(1);
    cout << t.first << endl;
    return 0;
}

但是，在初始化列表中，second 后于 first 进行初始化，导致 first 使用了未被初始化的 second 的值，从而引发未定义行为。

1.成员变量在使用初始化列表初始化时，与构造函数中初始化成员列表的顺序无关，只与定义成员变量的顺序有关

2.如果不使用初始化列表初始化，在构造函数内初始化时，此时与成员变量在构造函数中的位置有关。

B中有类A成员变量：

A构造调用 - B构造调用 - B析构调用 - A析构调用

1 基类的静态变量或全局变量

2 派生类的静态变量或全局变量

重载 #

1.无法被重载的运算符 #

在C++中，有一些运算符是无法被重载的。以下是无法被重载的运算符列表：

  1. 成员访问运算符：. 和 .*。
  2. 域解析运算符：::。
  3. 条件运算符（三元运算符都不能重载）：?:。
  4. 作用域运算符：sizeof。
  5. 类型运算符：typeid。
  6. 取地址运算符：&。
  7. 成员指针运算符：.* 和 ->*。

这些运算符具有特定的语法和语义，无法通过重载改变其行为。

2.一元运算符的正确重载 # 
/*
Obj Obj::operator+(const Obj& obj)
Obj Obj::operator+()
Obj Obj::operator<+>(const Obj& obj)
Obj Obj::operator(+)(const Obj& obj)
*/

一元运算符 + 的重载函数只有一个参数，即被操作的对象本身。因此，它不需要额外的参数来进行操作。重载函数的返回类型应该是 Obj 类型，表示对运算符操作后得到的新对象。

所以，正确的声明是：

Obj Obj::operator+();
3.有些运算符只能用成员函数进行重载 #
  • 二进制运算符=（赋值），[]（数组订阅），->（成员访问）以及n元()（函数调用）运算符必须始终实现为成员函数，因为语言的语法要求它们。

“=” 号 原因：在进行赋值的时候会调用构造函数，比如int a, a = 5. 会执行两次构造函数，如果没重载"="，会调用默认的；如果重载了会调用重载后的。如果对"=“用友元函数进行重载，友元函数只是类的“朋友”，不是类的成员函数，便会出现以下情景：程序会出现两种矛盾 1.认为没有重载赋值运算符的成员函数，调用相应的构造函数 2.在全局范围内，又重载了赋值运算符函数，赋值语句正好和这个函数匹配上了，也会去调用这个函数 出现了矛盾 ->、[]、()同样也是这样

4.«运算符只能用非成员函数进行重载 #

原因：运算符作为成员函数时，其左侧运算对象必须是运算符所属类的一个对象， 显然 “«” 不是

5.非成员函数重载运算符直接在类外部声明定义即可,也可以定义为类的友元函数 #

选择非成员函数重载运算符的另一个原因是为了使该运算符可交换。

如果我们将运算符重载为类的成员函数，那么操作数的顺序是固定的，左操作数是调用该成员函数的对象。但是，通过将运算符重载为非成员函数，我们可以在不同类型的操作数之间进行交换。

例如，对于二元运算符 +，我们可以定义两个重载函数：一个是成员函数形式的重载，另一个是非成员函数形式的重载。这样，我们既可以使用对象相加，也可以通过操作数顺序的交换来实现相同的结果。

cpp复制代码class MyClass {
public:
    int value;
    MyClass(int v) : value(v) {}

    // 成员函数形式的重载
    MyClass operator+(const MyClass& other) {
        return MyClass(value + other.value);
    }
};

// 非成员函数形式的重载
//MyClass为返回类型，有两个参数
MyClass operator+(const MyClass& obj1, const MyClass& obj2) {
    return MyClass(obj1.value + obj2.value);
}

通过这种方式，我们可以使用以下两种方式执行相同的操作：

cpp复制代码MyClass obj1(5);
MyClass obj2(10);

MyClass result1 = obj1 + obj2;       // 使用成员函数形式的重载
MyClass result2 = obj2 + obj1;       // 使用非成员函数形式的重载（交换了操作数顺序）

这种可交换性可以提高代码的灵活性和可读性。

a+b 	operator+(a,b) 	a.operator+(b)
-a 		operator-(a) 	a.operator-()
a++ 	operator++(a,0) a.operator++(0)
–a 		operator–(a) 	a.operator–();

左边为友元函数，右边为类成员函数
