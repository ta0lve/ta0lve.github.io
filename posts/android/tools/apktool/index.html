安装 #

Windows #

  1. 下载 Windows 包装脚本（右键单击，将链接另存为apktool.bat)
  2. 下载 最新的Apktool 。
  3. 将下载的 jar 重命名为apktool.jar
  4. 将两个文件 (apktool.jar & apktool.bat) 移动到Windows目录 (通常是 C://Windows目录)
  5. 如果您无权访问 C://Windows，则可以将这两个文件放在任何位置，然后将该目录添加到环境变量系统 PATH 变量中。
  6. 尝试通过命令提示符运行apktool
  7. 第一步包装器脚本也可以直接复制以下代码到apktool.txt再改名为apktool.bat文件
@echo off
setlocal
set BASENAME=apktool_
chcp 65001 2>nul >nul

set java_exe=java.exe

if defined JAVA_HOME (
set "java_exe=%JAVA_HOME%\bin\java.exe"
)

rem Find the highest version .jar available in the same directory as the script
setlocal EnableDelayedExpansion
pushd "%~dp0"
if exist apktool.jar (
    set BASENAME=apktool
    goto skipversioned
)
set max=0
for /f "tokens=1* delims=-_.0" %%A in ('dir /b /a-d %BASENAME%*.jar') do if %%~B gtr !max! set max=%%~nB
:skipversioned
popd
setlocal DisableDelayedExpansion

rem Find out if the commandline is a parameterless .jar or directory, for fast unpack/repack
if "%~1"=="" goto load
if not "%~2"=="" goto load
set ATTR=%~a1
if "%ATTR:~0,1%"=="d" (
    rem Directory, rebuild
    set fastCommand=b
)
if "%ATTR:~0,1%"=="-" if "%~x1"==".apk" (
    rem APK file, unpack
    set fastCommand=d
)

:load
"%java_exe%" -jar -Duser.language=en -Dfile.encoding=UTF8 -Djdk.util.zip.disableZip64ExtraFieldValidation=true -Djdk.nio.zipfs.allowDotZipEntry=true "%~dp0%BASENAME%%max%.jar" %fastCommand% %*

rem Pause when ran non interactively
for /f "tokens=2" %%# in ("%cmdcmdline%") do if /i "%%#" equ "/c" pause

Linux #

  1. 下载 Linux 包装器脚本（右键单击，将链接另存为apktool)

  2. 下载 最新的Apktool 。

  3. 将下载的 jar 重命名为apktool.jar

  4. 将两个文件 (apktool.jar & apktool) 移动到 /usr/local/bin` (需要 root 权限)

  5. Make sure both files are executable (chmod +x)

  6. 尝试通过 cli （command-line interface， 命令行界面）运行apktool

    • # 要求java的版本大于等于1.8
# 若之前没有配置过java环境则需要先安装java-1.8
sudo apt install openjdk-8-jre-headless

  7. 第一步包装器脚本也可以直接复制以下代码到apktool.txt再改名为apktool.bat文件

#!/bin/bash

# This script is a wrapper for smali.jar, so you can simply call "smali",
# instead of java -jar smali.jar. It is heavily based on the "dx" script
# from the Android SDK

# Set up prog to be the path of this script, including following symlinks,
# and set up progdir to be the fully-qualified pathname of its directory.
prog="$0"
while [ -h "${prog}" ]; do
    newProg=`/bin/ls -ld "${prog}"`

    newProg=`expr "${newProg}" : ".* -> \(.*\)$"`
    if expr "x${newProg}" : 'x/' >/dev/null; then
        prog="${newProg}"
    else
        progdir=`dirname "${prog}"`
        prog="${progdir}/${newProg}"
    fi
done
oldwd=`pwd`
progdir=`dirname "${prog}"`
cd "${progdir}"
progdir=`pwd`
prog="${progdir}"/`basename "${prog}"`
cd "${oldwd}"

jarfile=apktool.jar
libdir="$progdir"
if [ ! -r "$libdir/$jarfile" ]
then
    echo `basename "$prog"`": can't find $jarfile"
    exit 1
fi

javaOpts=""

# If you want DX to have more memory when executing, uncomment the following
# line and adjust the value accordingly. Use "java -X" for a list of options
# you can pass here.
#
javaOpts="-Xmx512M -Dfile.encoding=utf-8 -Djdk.util.zip.disableZip64ExtraFieldValidation=true -Djdk.nio.zipfs.allowDotZipEntry=true"

# Alternatively, this will extract any parameter "-Jxxx" from the command line
# and pass them to Java (instead of to dx). This makes it possible for you to
# add a command-line parameter such as "-JXmx256M" in your ant scripts, for
# example.
while expr "x$1" : 'x-J' >/dev/null; do
    opt=`expr "$1" : '-J\(.*\)'`
    javaOpts="${javaOpts} -${opt}"
    shift
done

if [ "$OSTYPE" = "cygwin" ] ; then
	jarpath=`cygpath -w  "$libdir/$jarfile"`
else
	jarpath="$libdir/$jarfile"
fi

# add current location to path for aapt
PATH=$PATH:`pwd`;
export PATH;
exec java $javaOpts -jar "$jarpath" "$@"

  6. 第一步包装器脚本也可以直接复制 Linux 的安装代码到apktool.txt再改名为apktool.bat文件

使用 #

Utility 实用选项 #

用于实用命令的选项 （apktool {options}）

-advance, --advanced
  • 转储高级使用情况输出。
-version, --version
  • 输出当前版本。（例如：1.5.2

decode 反编译选项 #

即我们常说的解包，简要说明：

apktool d[ecode] [options] <file_apk>

-f,–force ,覆盖之前的文件

-o,–output ,输出路径(dir)

-p,–frame-path ,指定 framework 文件路径（dir）

-r,–no-res , 不解释 resources 资源

-s,–no-src ,不把dex 解释成smail

-t,–frame-tag , framework 标签

详细使用说明：

用于反编译命令的选项（apktool d 文件.apk {选项}）

-api, --api-level <API>
  • 要生成的 smali 文件的数字 API 级别（默认为 targetSdkVersion）
-b, --no-debug-info
  • 防止 baksmali 写出调试信息（.local、.param、.line 等）。
-f, --force
  • 强制删除目标目录。
--force-manifest
  • 强制解码，而不考虑资源的解码参数。AndroidManifest.xml
--keep-broken-res
  • 如果出现错误并且某些资源被删除，例如“检测到无效的配置标志。丢弃资源“，但您无论如何都想解码它们，即使有错误。
  • 您必须在构建之前手动修复它们。
-m, --match-original
  • 匹配尽可能接近原始文件的文件，但代价是防止重建。
--no-assets
  • 防止解码/复制未知资产文件。
--only-main-classes
  • 仅在应用程序的根目录中反汇编 dex 类。
-p, --frame-path <DIR>
  • 应存储/读取框架文件的文件夹位置。
-r, --no-res
  • 这将防止资源的反编译。这保持resources.arsc
-s, --no-src
  • 这将防止 dex 文件的反汇编。这会保留 apk dex 文件，并在构建过程中简单地移动它。
-t, --frame-tag <TAG>
  • 使用由 标记的框架文件。TAG

build 回编译选项 #

其实就是把apk反汇编形成的目录回编译为apk，且得到的apk是未签名的apk

apktool b[uild] [options] <app_path>

-f,–force-all ,覆盖之前的文件

-o,–output ,输出路径(dir)

-p,–frame-path , 指定 framework 文件路径(dir)

# 例子：
apktool b <dir>  # apk文件默认在dir/dist目录下 
apktool b <dir> -o <file.apk>  # 自己指定apk位置

详细说明：

用于构建命令的选项（apktool b 文件夹 {选项}）

-a, --aapt <FILE>
  • 从指定位置加载 aapt/aapt2 二进制文件，而不是依赖于内部版本。
-api, --api-level <API>
  • 要针对其构建的 smali 文件的数字 API 级别（默认为minSdkVersion)
-c, --copy-original
  • 复制原件和 .AndroidManifest.xml``META-INF
-d, --debug
  • 添加到 。debuggable="true"``AndroidManifest.xml
-f, --force-all
  • 在生成过程中覆盖现有文件，包括资源和源。
-n, --net-sec-conf
  • 在 apk 中添加通用网络安全配置文件。
-nc, --no-crunch
  • 在生成期间禁用资源文件的处理。
-o, --output <FILE>
  • 写入的 apk 的名称。默认值为 。dist/{apkname}.apk
-p, --frame-path <DIR>
  • 存储/读取框架文件的位置。
--use-aapt2
  • 使用 aapt2 而不是 aapt。

空框架目录选项 #

用于 empty-framework-dir 命令的选项 （apktool empty-framework-dir {options}）

-f, --force
  • 强制删除目标目录。
-p, --frame-path <DIR>
  • 应存储/读取框架文件的文件夹位置。

列表框架目录选项 #

用于列表框架命令的选项（apktool list-frameworks {options}）

-p, --frame-path <DIR>
  • 应存储/读取框架文件的文件夹位置。

常用选项 #

可与任何命令一起使用的选项

-v, --verbose
  • 详细输出。包括记录器中标记的任何消息。FINE
-q, --quiet
  • 快速的输出。
